Компоненты 2.0::Форма ввода тегов

Компонент предназначен для вывода формы ввода тегов.

Пример вызова

<?$APPLICATION->IncludeComponent(
"bitrix:search.tags.input",
"",
Array(
	"PAGE_ELEMENTS" => "10", 
	"SORT_BY_CNT" => "Y", 
	"TEXT" => "", 
	"TMPL_IFRAME" => "Y", 
	"NAME" => "TAG", 
	"VALUE" => "", 
	"arrFILTER" => "all",
	)
);?>

Описание параметров

Основные параметры
NAME
Название формы ввода тегов. По умолчанию указано TAG.

VALUE
Содержимое формы ввода тегов по умолчанию.

arrFILTER
Параметр служит для ограничение области поиска: статическими файлами (main), либо форумами(forum), либо одним из типов инфоблоков, определенных в системе, либо блогами. Можно также выбрать несколько из перечисленных источников информации либо выбрать пункт Не ограничивать.
Если выбрать:
  1. [main] Cтатические файлы, то появляется для заполнения поле Путь к файлу начинается с любого из перечисленных, в котором можно указать путь к разделу или файлу, где будет осуществлен поиск, например, /about/. Если поле пусто, то поиск осуществляется во всех статических файлах системы.
  2. [forum] Форумы. Ниже отображается поле со всеми форумами, созданными на сайте. Среди них можно выбрать один или несколько форумов либо выбрать вариант (все), тогда поиск будет осуществлен во всех форумах системы.
  3. Один из типов инфоблоков (например, [iblock_news] Новости). При нажатии ОК станут доступны все инфоблоки, созданные внутри выбранного типа. Среди них можно выбрать один или несколько инфоблоков либо выбрать вариант (все), тогда поиск будет осуществлен во всех инфоблоках этого типа.
  4. [blog] Блоги. При данном ограничении поиск будет осуществлен только в рамках блогов.

Дополнительно
PAGE_ELEMENTS
Количество записей в выпадающем списке-подсказке уже существующих тегов.

SORT_BY_CNT
[Y|N] При отмеченной опции теги в списке-подсказке будут отсортированы по популярности. 

TEXT
Код дополнительного свойства.

TMPL_IFRAME
[Y|N] Вставлять фрейм в шаблон компонента.
