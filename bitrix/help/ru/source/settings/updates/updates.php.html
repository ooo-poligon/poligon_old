Обновление ядра продукта

Если доступны обновления ядра продукта, то на главной странице системы обновлений появляется ссылка на просмотр и загрузку этих обновлений.

Система обновлений производит технически сложную серьезную модификацию ядра продукта. Если эта модификация будет произведена с ошибками, то сайты, работающие на этом ядре, могут оказаться неработоспособными. Перед установкой обновлений рекомендуется убедиться в наличии резервной копии как базы данных, так и скриптов ядра продукта и служебной области. Рекомендуется для проведения процедуры обновления выбирать время, когда нагрузка на сервер минимальна. При возникновении проблем с установкой обновления вам необходимо незамедлительно обратиться в службу технической поддержки компании 1C-Битрикс.

Обратите внимание, что в случае медленного канала или большой загрузки сервера обновление ядра продукта может занять некоторое время.

Обновление ядра продукта осуществляется в несколько шагов: выбор требуемого действия (первый шаг), просмотр и загрузка выбранных обновлений (второй шаг), установка выбранных обновлений (третий шаг) и отчет об установке (четвертый шаг).

Выбор требуемого действия (первый шаг)

Этот шаг соответствует главной странице системы обновлений. Если доступны обновления ядра продукта, то на главной странице системы обновлений появляется ссылка на просмотр и загрузку этих обновлений (на второй шаг).

Просмотр и загрузка выбранных обновлений (второй шаг)

На этом шаге показывается список доступных обновлений, которые можно загрузить с сервера обновлений. По умолчанию будут загружены все доступные обновления. Если какое-либо из обновлений загружать не нужно, то можно убрать его из списка загрузки с помощью соответствующей кнопки "Удалить". Обновления, которые были убраны из списка загрузки, можно вернуть обратно в этот список с помощью соответствующей кнопки "Добавить". Обратите внимание, что в случае связанных обновлений в списке загрузки должны быть все связанные обновления либо не должно быть ни одного из них (система обновлений контролирует выполнение этого правила).

На этом шаге показывается так же общий размер выбранных для загрузки обновлений. Если размер обновлений велик и ваш хостинг не дает возможности поставить все обновления за один раз, то можно попробовать установить обновления по частям (по одному или нескольким модулям).

Внимание! Внимательно прочитайте описания обновлений модулей - они содержат важную информацию по установке обновления и предупреждают о возможных проблемах в работе.

Для перехода на третий шаг процедуры обновления ядра продукта необходимо нажать на кнопку "Далее". Это действие запустит механизм загрузки выбранных обновлений с сервера обновлений. Загрузка может занять некоторое время.

Просмотр и установка выбранных обновлений (третий шаг)

К этому шагу выбранные обновления загружены с сервера обновлений и готовы к установке. Страница показывает список доступных обновлений, которые можно установить. По умолчанию будут установлены все загруженные обновления. Если какое-либо из обновлений устанавливать не нужно, то можно убрать его из списка установки с помощью кнопки "Удалить". Обновления, которые были убраны из списка установки, можно вернуть обратно в этот список с помощью соответствующей кнопки "Добавить". Обратите внимание, что в случае связанных обновлений в списке установки должны быть все связанные обновления либо не должно быть ни одного из них (система обновлений контролирует выполнение этого правила).

Для начала установки необходимо нажать на кнопку "Установить". Это действие запустит механизм установки выбранных обновлений. Установка может занять некоторое время, которое зависит от объема обновления и текущей загруженности системы. Крайне рекомендуется для установки обновлений выбирать время, когда загруженность системы минимальна.

Отчет об установке (четвертый шаг)

На этом шаге выводится информация об итогах установки обновления и произошедших при установке ошибках.

Внимание! Не обновляйте эту страницу и не используйте кнопку "Назад" браузера.

Если страница сообщает о наличии каких-либо ошибок или если страница вообще не загрузилась корректно, вам необходимо незамедлительно проверить работоспособность сайта и выяснить причину возникновения ошибки. Если вы не можете сделать это самостоятельно, незамедлительно обратитесь в службу техподдержки компании 1С-Битрикс. Для ускорения решения проблемы мы рекомендуется приложить к обращению файл /bitrix/updates/updater.log.

Просмотреть результаты работы системы обновлений вы можете так же в журнале установки.
