RSS новости (экспорт)

Одностраничный компонент. Предназначен для создания страницы, которая отдает выбранные новости вашего сайта в формате RSS.

Пример вызова

<?$APPLICATION->IncludeComponent(
"bitrix:rss.out",
"",
Array(
	"IBLOCK_TYPE" => "news", 
	"IBLOCK_ID" => "33", 
	"SECTION_ID" => "", 
	"NUM_NEWS" => "20", 
	"NUM_DAYS" => "30", 
	"YANDEX" => "N", 
	"CACHE_TYPE" => "A", 
	"CACHE_TIME" => "3600" 
)
);?>

Описание параметров

Основные параметры
IBLOCK_TYPE
Тип информационного блока. Указывается один из созданных в системе типов.
IBLOCK_ID
Код информационного блока, информация из которого будет экспортироваться.
SECTION_ID
Код раздела. В этом случае будет экспортироваться информация только из этого раздела.
NUM_NEWS
Количество новостей для экспорта.
NUM_DAYS
Количество дней для экспорта.
YANDEX
Экспортировать в диалект Яндекса.
Настройки кеширования
CACHE_TYPE
Тип кеширования. Доступны следующие значения:
  • А - Автоматическое кеширование. Действует при включенном автокешировании на странице Настройки кеширования (Настройки > Настройки продукта > Автокеширование) и при установленном времени кеширования в поле CACHE_TIME (сек.).
  • Y - Кешировать . Время кеширования (CACHE_TIME) должно быть установлено в ручную. Настройки автокеширования в этом случае не работают.
  • N - Не кешировать . При выборе этого типа кеширование не происходит, независимо от настроек автокеширования (Настройки > Настройки продукта > Автокеширование)
CACHE_TIME
Время кеширования в секундах.
CACHE_FILTER
[Y|N] Кешировать при установленном фильтре. При установленной опции каждый результат, полученный из фильтра будет кешироваться. Имеет смысл отмечать, если многие пользователи используют одну и туже выборку из фильтра.
