Каталог::Разделы с TOP’ом элементов

Компонент предназначен для вывода TOP’а элементов каждого раздела в табличном порядке. Как правило используется на главной странице каталога.

Пример вызова

<?
$APPLICATION->IncludeFile("iblock/catalog/sections_top.php", Array(
"IBLOCK_TYPE"		=> "catalog",
"IBLOCK_ID"		=> "21",
"PARENT_SECTION_ID"	=> "0",
"SECTION_SORT_FIELD"	=> "sort",
"SECTION_SORT_ORDER"	=> "asc",
"SECTION_COUNT"            => "20",
"SECTION_URL"		=> "/catalog/phone/section.php?",
"ELEMENT_COUNT"            => "6",
"LINE_ELEMENT_COUNT"	=> "2",
"ELEMENT_SORT_FIELD"	=> "sort",
"ELEMENT_SORT_ORDER"	=> "asc",
"arrPROPERTY_CODE" => Array(
			"YEAR",
			"STANDBY_TIME",
			"TALKTIME",
			"WEIGHT",
			"STANDART",
			"SIZE",
			"BATTERY"
			),
"PRICE_CODE"		=> "RETAIL",
"BASKET_URL"		=> "/personal/basket.php",
"FILTER_NAME"		=> "arrFilter",
"CACHE_FILTER"		=> "N",
"CACHE_TIME"		=> "3600",
"DISPLAY_PANEL"		=> "Y",
"ACTION_VARIABLE"		=> "action",
"PRODUCT_ID_VARIABLE"	=> "id",
"SHOW_DESCRIPTION"		=> "N",
"USE_PRICE_COUNT"		=> "N",
"SHOW_PRICE_COUNT"		=> "1",
));
?>

Описание параметров

IBLOCK_TYPE
Тип инфоблока.

IBLOCK_ID
ID инфоблока.

PARENT_SECTION_ID
ID родительского раздела.

SECTION_SORT_FIELD
Сортировка может быть выполнена по:
  • sort – по значнию индекса сортировки;
  • timestamp_x – по дате модификации параметров раздела;
  • name – по загол,овку раздела;
  • id – по ID разделу;
  • depth_level – уровень вложенности раздела.

SECTION_SORT_ORDER
Сортировка может быть выполнена по:
  • asc – по возрастанию;
  • desc – по убыванию.

SECTION_COUNT
Максимальное количество выводимых разделов.

SECTION_URL
URL, ведущий на страницу с содержимым раздела.

ELEMENT_COUNT
Максимальное количество элементов, выводимых в каждом разделе.

LINE_ELEMENT_COUNT
Количество элементов выводимых в одной строке таблицы.

ELEMENT_SORT_FIELD
По какому полю сортировать элементы; доступны следующие значения:
  • shows – среднее количество просмотров элемента (популярность элемента);
  • sort – по индексу сортировки;
  • timestamp_x – по дате изменения;
  • name – по заголовку;
  • id – по ID элемента;
  • active_from – по дате активности “с”;
  • active_to – по дате активности “по”.

ELEMENT_SORT_ORDER
Порядок сортировки элементов, допустимы следующие значения:
  • asc – по возрастанию;
  • desc – по убыванию.

arrPROPERTY_CODE
Массив символьных кодов свойств раздела.

PRICE_CODE
Символьный код типа цены.

BASKET_URL
URL, ведущий на страницу с корзиной покупателя.

FILTER_NAME
Имя массива со значениями фильтрации элементов.

CACHE_FILTER
[Y|N] - кешировать ли значения, выбираемые из базы, если установлен фильтр по ним.

CACHE_TIME
(сек.) время кеширования значений выбираемых из базы.

DISPLAY_PANEL
Добавлять в административную панель кнопки для данного компонента.

ACTION_VARIABLE
Название переменной, в которой передается действие, например, BUY и др.

PRODUCT_ID_VARIABLE
Название переменной, в которой передается код товара для покупки.

SHOW_DESCRIPTION
Показывать описание для списка.

USE_PRICE_COUNT
Использовать вывод цен с диапазонами. При отмеченной опции выводятся цены всех типов на товары.

SHOW_PRICE_COUNT
Указывается, для какого количества товаров выводить цены.
